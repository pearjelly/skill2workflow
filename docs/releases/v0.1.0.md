# skill2workflow v0.1.0 Release Notes

`v0.1.0` is the first open-source bootstrap release of `skill2workflow`.

It proves the core product direction: standard Agent `SKILL.md` files can be compiled into controlled workflows with validation, visualization, durable execution, pause/resume, connector execution, and auditability.

## Release Scope

This release includes:

- Skill parser for frontmatter, hard gates, checklist steps, and source line metadata
- Workflow DSL `0.1.0` compiler and structured validator
- Versioned JSON Schema at `schemas/workflow.schema.json`
- Local executor with JSON and SQLite run state
- Human-gate pause/resume
- Run summaries and run detail inspection
- LiteGraph-compatible visualization JSON
- Static LiteGraph editor with safe write-back for allowlisted authoring fields
- Immutable local control-plane publish and version registry
- Published workflow runs, resume, and audit filters
- JSON/JSONL and SQLite control-plane metadata storage
- Built-in manual and HTTP connector runtime MVP
- Example workflows for approval and HTTP connector scenarios
- Contributor guide, issue templates, compatibility notes, and stability boundaries

## Install And Verify

From a fresh checkout:

```bash
PYTHONPATH=src python3 -m unittest discover -s tests -v
PYTHONPATH=src python3 -m skill2workflow.cli compile examples/skills/approval-flow/SKILL.md -o /tmp/skill2workflow-workflow.json
PYTHONPATH=src python3 -m skill2workflow.cli validate /tmp/skill2workflow-workflow.json --format json
PYTHONPATH=src python3 -m skill2workflow.cli run /tmp/skill2workflow-workflow.json --state-dir /tmp/skill2workflow-state
```

## Compatibility

- Python: 3.9+
- Workflow DSL schema version: `0.1.0`
- Runtime dependencies: Python standard library
- Package version: `0.1.0`

The compatibility policy is documented in `docs/workflow-dsl-compatibility.md`.

## Stable In This Release

- Workflow DSL `0.1.0` top-level contract
- Structured validation error shape
- CLI commands listed in `README.md` and `HARNESS.md`
- JSON and SQLite local persistence modes
- Published workflow artifact immutability
- Example workflow fixture validity

## Experimental In This Release

- Parser heuristics for arbitrary real-world `SKILL.md` styles
- LiteGraph editor UI and visual write-back coverage
- Connector manifest shape and HTTP connector request metadata
- Control-plane storage layout
- Executor event names beyond documented audit/event examples

See `docs/stability.md` for detailed boundaries.

## Not Included

- Cloud-hosted control plane
- Full RBAC or IAM
- Enterprise connector credential management
- Distributed scheduling
- Complete BPMN compatibility
- Guaranteed automatic conversion of arbitrary SOP documents

## Upgrade Notes

This is the first release. There is no migration from an earlier public release.

Future `0.1.x` releases should keep Workflow DSL `0.1.0` readable unless a documented migration path is provided.
